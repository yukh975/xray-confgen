# История изменений

## v1.0.4 — 2026-03-22

### Интерфейс
- Раздел VLESS теперь поддерживает несколько записей — каждая с необязательным полем **Имя** и URI; кнопка **+ Добавить VLESS** добавляет ещё один сервер, ✕ удаляет запись
- Добавлен раздел **Balancer** между VLESS и Sniffing: галочка включения, выбор стратегии (random / leastPing), URL и интервал проверки observatory (отображаются только для leastPing)
- Список действий в правилах маршрутизации и селектор **Маршрут по умолчанию** теперь динамические: отражают текущий набор VLESS-тегов (и включают вариант `balancer` при включённом балансировщике)
- Подсказка в поле имени VLESS показывает автоматически генерируемый тег (proxy, proxy2, …)
- Метки над полями Имя и URI выровнены по левому краю полей ввода
- Кнопка удаления (✕) выровнена по вертикали с полем URI, а не с верхом строки
- **Обнаружение дублирующихся URI**: при вводе одинакового URI в двух строках дубликат подсвечивается красным и сразу выводится ошибка
- **Валидация URI при генерации**: каждое поле VLESS должно содержать непустой URI, начинающийся с `vless://`; некорректные поля выделяются красным, генерация блокируется

### Безопасность
- Во все PHP API-эндпоинты добавлены заголовки `X-Content-Type-Options: nosniff` и `X-Frame-Options: DENY`
- Inline-скрипт инициализации темы вынесен в `assets/theme-init.js`, что позволяет задать строгий `Content-Security-Policy: script-src 'self'` в nginx
- Пример конфигурации nginx обновлён: добавлены заголовки безопасности (`X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Content-Security-Policy`) и ограничение частоты запросов (`limit_req_zone`, 20 r/s, burst 30)
- Зарезервированные имена тегов (`direct`, `block`, `balancer`) теперь запрещены: в UI — в реальном времени с красной подсветкой, в PHP-бэкенде — с ошибкой

### Функционал
- **Несколько outbound**: каждая VLESS-запись становится отдельным outbound в конфиге; теги берутся из поля Имя или назначаются автоматически (proxy, proxy2, …)
- **Балансировщик**: при включении в раздел routing добавляется блок `balancers` с выбранной стратегией; правила маршрутизации и catch-all могут направлять трафик в outbound `balancer`; включение при менее чем 2 VLESS-записях заблокировано с показом ошибки
- **Стратегия leastPing**: при выборе в конфиг добавляется блок `observatory` с настроенным URL и интервалом проверки
- **Импорт config.json** обновлён: все VLESS-outbound импортируются как отдельные записи; настройки балансировщика и observatory восстанавливаются
- Пост-квантовое шифрование: поля `encryption=mlkem768x25519plus...` и `pqv` парсятся из VLESS URI и включаются в user-запись outbound; корректно восстанавливаются при импорте config.json
- **Модал «Поделиться»**: нажатие кнопки открывает модальное окно с QR-кодом (сжатым через deflate для читаемости) и полной ссылкой на конфигурацию, отображённой на экране; кнопка «Копировать» позволяет скопировать ссылку из модала; если ссылка слишком длинная для QR (например, из-за PQ-ключа) — показывается предупреждение, ссылка по-прежнему доступна
- **Вставить из QR**: в каждой строке VLESS появилась кнопка «Вставить из QR» — на мобильном открывает камеру, на компьютере — выбор файла; изображение QR декодируется через jsQR и URI вставляется в поле
- Мультивыбор (правила маршрутизации, DNS-правила): выбранные пункты отображаются вверху выпадающего списка, отсортированные по алфавиту; пересортировка при каждом изменении выбора
- **Удаление DNS-сервера**: удаление сервера, используемого в правилах, заблокировано с сообщением «Сначала удалите или измените эти правила»; индексы в правилах с более высокими номерами автоматически пересчитываются
- Правила маршрутизации по умолчанию упрощены: удалены `geosite:ru → direct` и `geoip:ru → direct`; теперь только `geoip:private → direct` и `geosite:category-ads-all → block`
- Кнопки «Добавить правило» и «Очистить правила» в разделе DNS выровнены по правому краю
- Блок Sniffing теперь всегда развёрнут по умолчанию — при первом открытии и после нажатия «Очистить»

## v1.0.3 — 2026-03-21

### Интерфейс
- Кнопка переключения тёмной/светлой темы (иконка солнца/луны) в заголовке; по умолчанию тёмная тема
- Выбранная тема сохраняется в `localStorage` и восстанавливается без мерцания
- Кнопка **«Пресеты»** в разделе маршрутизации открывает выпадающий список с мультивыбором: Россия, Иран, Блокировать рекламу, Всё через прокси, Блокировать BitTorrent
  - Пресеты аддитивны — добавляют правила, не удаляя существующие; дубликаты пропускаются
  - Одновременно можно активировать несколько пресетов; снятие галочки удаляет только правила этого пресета
  - Правила пресета фильтруются по базам данных, доступным на сервере
- **«Блокировать BitTorrent»** перенесено из отдельной галочки в выпадающий список пресетов
- Кнопка **«Очистить правила»** добавлена в разделы маршрутизации и DNS
- Расположение кнопок в разделе маршрутизации: «Пресеты» слева, «Добавить правило» и «Очистить правила» справа; аналогично в DNS-правилах
- Кнопка **«Поделиться»** кодирует всё состояние формы в URL-безопасный base64-параметр (`?s=…`); ссылку можно открыть на любом устройстве для восстановления конфигурации; параметр очищается из адресной строки после восстановления
- «VLESS URL» переименован в «VLESS URI» везде: метка поля, подзаголовок, справка

### Функционал
- Добавлен раздел **Mux** между DNS и Logging: галочка включения (по умолчанию выключен), `concurrency`, `xudpConcurrency`, `xudpProxyUDP443`; автоматически игнорируется при использовании Reality + `xtls-rprx-vision` flow
- **HTTP inbound**: кнопка в разделе Inbound добавляет HTTP-прокси inbound (по умолчанию `127.0.0.1:8080`); IP и порт настраиваются; кнопка ✕ удаляет inbound; поддерживается только один HTTP inbound
- Добавлен раздел **Sniffing** между VLESS и Databases: галочка включения (по умолчанию включён), чекбоксы протоколов (`http`, `tls`, `quic`, `bittorrent`) и опция `routeOnly`; конфиг sniffing генерируется только при включённом разделе
- Кнопка **«Импортировать config.json»**: загрузка существующего файла `config.json` обратно в форму — все поля (inbound, VLESS URI, правила маршрутизации, DNS, журналирование) заполняются автоматически; после этого конфигурацию можно скорректировать и сгенерировать заново
- **Защита от дублирования DNS-серверов**: каждый пресет можно добавить только один раз; кнопка *Добавить сервер* автоматически выбирает первый незанятый пресет; при попытке вручную выбрать уже существующий пресет выводится ошибка и выбор откатывается
- **Адаптивная вёрстка для мобильных**: интерфейс корректно отображается на смартфонах — поля складываются вертикально, строки правил переходят в двухстрочный формат, touch-цели увеличены, автозум iOS отключён

### Исправления
- Пикер тегов показывал устаревшие теги после смены базы данных: состояние гонки приводило к тому, что завершившийся `fetchTags()` перезаписывал `knownTags` уже после вызова `resetPicker()`; исправлено счётчиком последовательности, инвалидирующим устаревшие запросы

## v1.0.2 — 2026-03-21

### Интерфейс
- Кнопка показа пароля использует SVG-иконки (открытый / перечёркнутый глаз) с переключением по клику
- Именованные заголовки разделов: Inbound, VLESS, Databases, Routing, Logging
- Одинаковые вертикальные отступы между всеми разделами формы
- Упрощены метки полей: убрано слово «Inbound» из меток IP и порта
- Метки полей выровнены по горизонтали с самими полями
- Упрощены тексты подсказок
- Заголовок страницы и H1 обновлены: «Xray config generator»
- Проект переименован из «vless-parser» в «xray-confgen» (Xray config generator)
- Раздел маршрутизации теперь сворачивается (по умолчанию скрыт); все необязательные разделы свёрнуты при первом открытии
- Результат `config.json` выводится во всплывающем модальном окне по центру экрана с фоном и кнопкой закрытия
- Окно результата закрывается только кнопкой ✕; клик по фону и Escape его не закрывают
- Клавиша Escape закрывает окно справки и окно ошибки
- В выпадающем списке выбора тегов добавлена кнопка «очистить всё» ✕ (отображается только при наличии выбранных элементов)
- Разделы DNS, маршрутизации и журналирования имеют единый паттерн «галочка → разворачивается»
- Добавлена кнопка справки (?) рядом с переключением языка; открывает модальное окно с руководством пользователя и переключателем языка

### Функционал
- **Добавлен раздел DNS**:
  - Галочка включения со сворачиваемым блоком
  - Список DNS-серверов: пресеты (Google DoH, Cloudflare DoH, Yandex DoH, Google DNS, Cloudflare DNS, Yandex DNS) или свой с именем и адресом
  - Правила DNS: привязка тегов из геобаз к конкретным серверам (тот же пикер, что и в маршрутизации)
  - Резервный DNS-сервер, стратегия запросов (UseIP / UseIPv4 / UseIPv6 / ForceIP / ForceIPv4 / ForceIPv6 / useSystem), стратегия доменов (AsIs / IPIfNonMatch / IPOnDemand)
  - Все DoH-пресеты используют IP-адреса, чтобы не зависеть от начального DNS-резолвера
- **Маршрутизация**: конфиг routing генерируется только когда раздел включён
- Строка кастомного DNS-сервера содержит поле имени, используемое как метка в селекторе правил

### Исправления
- Селектор сервера в правилах DNS отображал буквальный ключ перевода вместо названия сервера
- Пикер в правилах DNS всегда использовал исходную базу данных вместо текущей
- Выпадающий список пикера перекрывался разделом журналирования ниже (`overflow: hidden` → `overflow: visible`)

### Интерфейс
- Добавлен favicon: тёмный фон, тонкая синяя X с 8 точками-лучами (16×32×48 px)
- SVG-логотип добавлен слева от заголовка страницы
- В футер добавлена ссылка «Source code on GitHub»

### Документация
- README переработан: раздел Features заменён полным руководством пользователя (из окна справки)
- Добавлена ссылка на демо: https://yukh.net/xray-confgen/
- SVG-логотип добавлен в заголовки README
- В раздел установки добавлены ссылки для скачивания геобаз

## v1.0.0 — 2026-03-19

Первый релиз.

### Функционал
- Парсинг VLESS URI с автоматическим определением транспорта и типа безопасности
- Поддерживаемые транспорты: TCP, XHTTP (SplitHTTP), WebSocket, gRPC, HTTP/2
- Поддерживаемая безопасность: Reality, TLS
- SOCKS5 inbound с настраиваемым IP и портом
- Динамические правила маршрутизации с выбором базы геоданных
- Загрузка категорий прямо из `.dat` файлов через PHP protobuf-парсер
- Мультиселект с поиском и добавлением произвольного значения
- Отмеченные категории выводятся первыми (по алфавиту), затем остальные
- Сохранение состояния формы в `localStorage`
- Кнопки скачивания и копирования результата

### Базы данных
- В составе репозитория: `geosite.dat`, `geoip.dat`, `geosite_RU.dat`, `geoip_RU.dat`
- Автоматическое определение файлов из каталога `db/` при запуске

### Безопасность
- Серверная валидация IP через `FILTER_VALIDATE_IP`
- Белый список имён `.dat` файлов
- Защита от XSS: пользовательские данные выводятся через `textContent` / DOM-методы
