## v1.0.2 — 2026-03-21

### Интерфейс
- Кнопка показа пароля использует SVG-иконки (открытый / перечёркнутый глаз) с переключением по клику
- Именованные заголовки разделов: Inbound, VLESS, Databases, Routing, Logging
- Одинаковые вертикальные отступы между всеми разделами формы
- Упрощены метки полей: убрано слово «Inbound» из меток IP и порта
- Метки полей выровнены по горизонтали с самими полями
- Упрощены тексты подсказок
- Заголовок страницы и H1 обновлены: «Xray config generator»
- Проект переименован из «vless-parser» в «xray-confgen» (Xray config generator)
- Раздел маршрутизации теперь сворачивается (по умолчанию скрыт); все необязательные разделы свёрнуты при первом открытии
- Результат `config.json` выводится во всплывающем модальном окне по центру экрана с фоном и кнопкой закрытия
- Окно результата закрывается только кнопкой ✕; клик по фону и Escape его не закрывают
- Клавиша Escape закрывает окно справки и окно ошибки
- В выпадающем списке выбора тегов добавлена кнопка «очистить всё» ✕ (отображается только при наличии выбранных элементов)
- Разделы DNS, маршрутизации и журналирования имеют единый паттерн «галочка → разворачивается»
- Добавлена кнопка справки (?) рядом с переключением языка; открывает модальное окно с руководством пользователя и переключателем языка

### Функционал
- **Добавлен раздел DNS**:
  - Галочка включения со сворачиваемым блоком
  - Список DNS-серверов: пресеты (Google DoH, Cloudflare DoH, Yandex DoH, Google DNS, Cloudflare DNS, Yandex DNS) или свой с именем и адресом
  - Правила DNS: привязка тегов из геобаз к конкретным серверам (тот же пикер, что и в маршрутизации)
  - Резервный DNS-сервер, стратегия запросов (UseIP / UseIPv4 / UseIPv6 / ForceIP / ForceIPv4 / ForceIPv6 / useSystem), стратегия доменов (AsIs / IPIfNonMatch / IPOnDemand)
  - Все DoH-пресеты используют IP-адреса, чтобы не зависеть от начального DNS-резолвера
- **Маршрутизация**: конфиг routing генерируется только когда раздел включён
- Строка кастомного DNS-сервера содержит поле имени, используемое как метка в селекторе правил

### Исправления
- Селектор сервера в правилах DNS отображал буквальный ключ перевода вместо названия сервера
- Пикер в правилах DNS всегда использовал исходную базу данных вместо текущей
- Выпадающий список пикера перекрывался разделом журналирования ниже (`overflow: hidden` → `overflow: visible`)

### Интерфейс
- Добавлен favicon: тёмный фон, тонкая синяя X с 8 точками-лучами (16×32×48 px)
- SVG-логотип добавлен слева от заголовка страницы
- В футер добавлена ссылка «Source code on GitHub»

### Документация
- README переработан: раздел Features заменён полным руководством пользователя (из окна справки)
- Добавлена ссылка на демо: https://yukh.net/xray-confgen/
- SVG-логотип добавлен в заголовки README
- В раздел установки добавлены ссылки для скачивания геобаз

- Выбор маршрута по умолчанию (proxy / direct, по умолчанию: proxy)
- Выбор стратегии доменов (IPIfNonMatch / IPOnDemand / AsIs, по умолчанию: IPIfNonMatch)
- Раздел журналирования: включение, каталог, уровень (debug / info / warning / error / none)
- Выбранный язык сохраняется в `localStorage`

### Базы данных
- Добавлены базы для Ирана: `geoip_IR.dat`, `geosite_IR.dat`

### Код
- Все комментарии и сообщения об ошибках переведены на английский
- Русские строки интерфейса вынесены в отдельный файл `assets/translations.js`

## v1.0.0 — 2026-03-19

Первый релиз.

### Функционал
- Парсинг VLESS URI с автоматическим определением транспорта и типа безопасности
- Поддерживаемые транспорты: TCP, XHTTP (SplitHTTP), WebSocket, gRPC, HTTP/2
- Поддерживаемая безопасность: Reality, TLS
- SOCKS5 inbound с настраиваемым IP и портом
- Динамические правила маршрутизации с выбором базы геоданных
- Загрузка категорий прямо из `.dat` файлов через PHP protobuf-парсер
- Мультиселект с поиском и добавлением произвольного значения
- Отмеченные категории выводятся первыми (по алфавиту), затем остальные
- Сохранение состояния формы в `localStorage`
- Кнопки скачивания и копирования результата

### Базы данных
- В составе репозитория: `geosite.dat`, `geoip.dat`, `geosite_RU.dat`, `geoip_RU.dat`
- Автоматическое определение файлов из каталога `db/` при запуске

### Безопасность
- Серверная валидация IP через `FILTER_VALIDATE_IP`
- Белый список имён `.dat` файлов
- Защита от XSS: пользовательские данные выводятся через `textContent` / DOM-методы
